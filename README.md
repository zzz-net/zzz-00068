# 🏥 日间病房床位周转板

本地运行的日间病房床位周转管理系统，**无需对接医院系统**，所有数据本地持久化（localStorage）。支持床位可视化看板、预约登记、入出床流程、异常拦截、操作审计、数据导出。

---

## 📦 技术栈

| 类别 | 技术 |
|------|------|
| 框架 | React 18 + TypeScript 5 |
| 构建 | Vite 6 |
| 状态 | Zustand 5（带 persist 中间件自动持久化到 localStorage） |
| 样式 | TailwindCSS 3 + 医疗蓝主题 |
| 路由 | React Router DOM 7 |
| 图标 | Lucide React |

## 🔁 主流程复现步骤

### ✅ 流程一：正常预约 → 入床 → 护理 → 出床 → 导出

| 步骤 | 操作页面 | 具体操作 | 预期结果 |
|------|----------|----------|----------|
| 1 | `/login` | 选「李高级」→ 密码 `123456` → 登录 | 跳转 `/dashboard`，显示 12 张床位 |
| 2 | `/appointments/new` | 点击右上「+新建预约」→ 填患者：测试男/35岁/手机13800000001 → 诊断：高血压复查 → 无传染病 → 选床位 A-03 → 日期：今日 → 时段：下午 → 开始14:00/结束16:00 → 提交 | Toast「预约创建成功」，跳转回预约列表，新增一条 pending 预约 |
| 3 | `/appointments` | 找到刚创建的预约 → 点「确认入床」 | Toast「入床成功」，床位 A-03 卡片变为蓝色「占用」状态，状态变为 admitted |
| 4 | `/dashboard` → 点击 A-03 床位 | 在「当前状态」Tab → 「添加护理备注」→ 类型=给药，内容=「硝苯地平 10mg 口服」→ 确认 | 时间轴新增一条备注，操作日志记录 |
| 5 | 同抽屉 → 「正常出床」 | 弹确认 → 确认 | Toast「出床成功」，床位变为灰色「待清理」，appointment 状态 completed |
| 6 | 同抽屉 → admin 登录可见「标记已清理」 | 点击 | 床位变为绿色「空闲」，可接受下一位 |
| 7 | 顶部右上「导出」按钮 | 点击下载 `daily-report-{今日日期}.csv` | Excel 可打开，含床位号/患者/入床/出床/时长/护理次数等 13 列 |

### 🚫 流程二：非法链路拦截（**失败后原记录不变**）

#### 拦截场景 1：普通护士强制释放

| 步骤 | 操作 | 预期 |
|------|------|------|
| 1 | 登录「赵普通」(normal 角色) | 正常进入看板 |
| 2 | 点击任意「占用」床位（如 A-01）→ 「强制释放」 | Toast 红色错误：**「普通护士无权强制释放占用床位，请联系管理员或高级护士」** |
| 3 | 查看该床位 + `/history` 异常记录 | ✅ 床位状态仍为「占用」（原记录不变）；✅ 操作日志新增 isAbnormal=true；✅ 异常记录列表新增 1 条 `FORCE_RELEASE_DENIED` 未处理 |

#### 拦截场景 2：预约时段重叠

| 步骤 | 操作 | 预期 |
|------|------|------|
| 1 | 记住任意已有 pending 预约的 床位+时段 | 如 A-02 今日 14:00-16:00 |
| 2 | 新建预约 → 同床位 A-02 → 同时段 14:30-15:30 | 表单显示**红色警告框**高亮重叠冲突 |
| 3 | 强行提交（假设关闭了前端校验直接调用 store） | Store 返回 `{success:false, error:'该时段与已有预约重叠'}`，**预约数组长度不变（原记录不变）**，异常记录新增 1 条 `TIME_OVERLAP` |

#### 拦截场景 3：出床时间早于入床时间（直接模拟 dischargedAt 篡改）

| 步骤 | 操作 | 预期 |
|------|------|------|
| 1 | 找到任意 in_bed 的 admissionId（如 A-01） | 记录 admittedAt = 14:00 时间戳 |
| 2 | 在 BedDetailDrawer 中，若强制传入 dischargedAt = 10:00（早于入床） | Store 层校验拦截，返回 **「出床时间不能早于入床时间」**，床位仍为占用，异常记录新增 `DISCHARGE_BEFORE_ADMIT` |

## 🛡️ 数据安全

- **纯本地运行**：所有数据只存浏览器 localStorage，不上传任何服务器
- **导入校验**：恢复备份前校验 JSON schema，非法数据拒绝导入
- **原子操作**：核心写操作（预约创建/出床等）**先全量校验再写入**，任一步失败回滚，确保原记录不变
- **操作审计**：所有写操作留痕（操作人+时间+审批人），异常操作额外记录

## ❓ 常见问题

**Q: 样例数据导入后想清空重来？**
A: 管理员登录 → `/config` → Tab5「数据管理」→「清空全部数据」→ 二次确认后清空。

**Q: 忘记密码？**
A: 管理员登录 → `/config` → Tab4「护士角色」→ 对应护士行「重置密码」→ 密码重置为 `123456`。

**Q: 构建后部署？**
A: `npm run build` 产物在 `dist/` 目录，可直接放任意静态服务器（Nginx / IIS / GitHub Pages），无需后端。
